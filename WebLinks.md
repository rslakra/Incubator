- How to configure jetty server in spring-boot application?
  we can use jetty instead of tomcat in spring-boot-starter-web, by removing the existing dependency and including the following:
  <dependency>
  <groupId>org.springframework.boot</groupId>
  <artifactId>spring-boot-starter-web</artifactId>
  <exclusions>
  <exclusion>
  <groupId>org.springframework.boot</groupId>
  <artifactId>spring-boot-starter-tomcat</artifactId>
  </exclusion>
  </exclusions>
  </dependency>
  <dependency>
  <groupId>org.springframework.boot</groupId>
  <artifactId>spring-boot-starter-jetty</artifactId>
  </dependency>

Transition from Java to Python

Java -> Python (Programming Language)
SpringBoot -> Django or Flask (Web Framework)
Tomcat -> Gunicorn or uWSGI (WSGI HTTP Server)
Jetty -> Twisted or Tornado (Asynchronous Server)
Maven/Gradle -> pip (Package Installer) and setuptools (Library to Define Projects) or Poetry (Dependency Management)

--------------------------------------------------

Although there are ways to make fully self contained python applications that just run.  Pretty much all of them have various issues which requires a user to be aware of the limitations of them.
Specifically PyInstaller https://pyinstaller.org/en/stable/ will generate a self contained executable for a Python application as long as the application and it's dependencies follow Python best practices.  It also can require additional effort if the application or dependencies rely on Operating system components.  If the requirements aren't met don't it can be difficult to get things working.
Appimage https://appimage.org/ is another way to make a self contained executable for a Python application but it uses fuse support in the Operating system which can prevent it from working in containers without the necessary permissions.
The recommendation for a web service component is to use uwsgi.  We recommend using the Django framework for the web framework.
As for packaging we recommend using pip/setuptools which are a standard component of Python interpreters.   The handling and management of dependencies is well defined.  It is important to understand the difference between abstract and concrete dependencies.
